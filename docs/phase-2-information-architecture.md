# Phase 2 信息架构

## 路由

| 路由 | 作用 | 状态读写 |
|---|---|---|
| `/` | 3秒理解测试、开始入口、16型与研究入口 | 开始时创建本地会话 |
| `/test` | 前11题，一题一屏，聊天场景呈现 | 读写答案与题号 |
| `/update` | 四阶段平台更新冲击与第12题 | 读写更新选择、结果、完成状态 |
| `/result` | 三屏截图友好结果档案 | 读取答案与结果 |
| `/feedback` | 低保真试玩遗留的可选本地反馈 | 只写本机，可主动导出 |
| `/types` | 16型网格与四维口语筛选 | 无测试状态写入 |
| `/types/[code]` | 16个可静态生成的类型详情 | 无测试状态写入 |
| `/research` | 普通人可读的研究解释、边界与投票 | 可选本地入口计数 |
| `/about` | 项目、免责声明、隐私、数据、版本与预留信息 | 可关闭本地统计 |
| `/demo` | 大会快捷演示目录 | 不读写普通测试状态 |
| `/demo/update` | 独立平台更新演示 | 不读写普通测试状态 |
| `/debug` | 开发评分轨迹 | 生产环境404 |

## 主流程

`/` → `/test`（q01—q11）→ `/update`（熟悉、迁移、陌生、选择）→ `/result`。刷新后从第一道未答题恢复；返回修改会清除旧结果语义并重新计算。离开页面不会出现挽留弹窗。

三组 COMFORT/RELIABILITY 配对分别显示“此刻你更想收到哪一句？”与“冷静下来后，你觉得哪一句更值得相信？”。界面不公开 gap 计算；结果页才揭示背离次数。

## 结果信息层级

1. AITI代码、中文名、角色、原始定义、结果标题（若已确认）、哄感等级。
2. 致命AI台词、平台更新暴击、舒服—可靠背离、毒舌判词和等级说明。
3. 安全彩蛋、研究概念与统一免责声明。

正式分享卡仍属于 Phase 3；Phase 2 只提供长截图友好布局与打印/存PDF入口。
