# Phase 2.2 Multi-Option Question Redesign 复核

最终状态：工程完成，内容继续为 `DRAFT_REQUIRES_HUMAN_REVIEW`。

## 完成范围

- 12个情境改为6道三选、4道四选、2组舒服—可靠双选，共14次选择。
- 原人工修订情境作为最新来源；q10/q11明确标记为新增草案，没有伪装成原文确认内容。
- 每项代表不同的行为策略；无“以上都不是”、随机分型、动态阈值或人格配额。
- 每题只有一个主维度。双选每次+1，普通题主维度+2；第一轮审计后移除非必要次维度票，避免跨维累积偏差。
- 输出新增 `dimensionMargins`、`dimensionConfidence`、`primaryType`、`shadowType`；保留 `typeCode` 作为primaryType兼容别名。
- shadowType只翻转绝对margin最小的一维，不读取类型频率。
- 哄感范围重校准为raw 0—31与六段新阈值。
- `/test`纵向渲染3—4个选项，四选不使用2×2小按钮；选择反馈、返回改答、刷新恢复和reduced-motion继续保留。

## 评分边界

- 四维每极理论0—6，margin -6—+6，confidence 0—1。
- VF/OP零margin由该维最后一道题的实际选择裁决；仍相同时固定顺序只用于shadow margin并列。
- 16型理论比例5.36%—7.21%，不宣称真人等概率。
- 六级比例0.40%、12.30%、47.87%、34.73%、4.64%、0.07%；比例只反映均匀组合结构。
- Lv.5需raw≥26且必须多题累积；Lv.0需raw≤5且必须在多个场景保留边界。

## 审计与测试

- 精确DP覆盖15,116,544个组合与356,076个评分状态。
