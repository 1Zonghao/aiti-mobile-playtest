# Phase 1.5C 真人试玩指南

适用范围：5—15 名真人的移动端低保真试玩。题目与哄感公式仍为 `DRAFT_REQUIRES_HUMAN_REVIEW`。本轮只检查题目体验、评分流程、完成时长、结果感受和传播意愿，不产生科学研究结论。

## 本地启动

环境要求：Node.js 20.9 或更高版本，推荐使用仓库当前已验证的 Node.js 24。

```bash
npm install
npm run dev
```

电脑浏览器访问 `http://localhost:3000`。开发服务器会监听全部网络接口，以便同一局域网的手机访问。

## 让同一局域网手机访问

1. 确认电脑与手机连接同一个可信 Wi-Fi。
2. 在 PowerShell 执行 `ipconfig`，找到当前 Wi-Fi 网卡的 IPv4 地址，例如 `192.168.1.23`。
3. 手机浏览器访问 `http://192.168.1.23:3000`，把示例地址替换成电脑实际 IPv4。
4. 如果打不开，允许 Windows 防火墙放行 Node.js 的“专用网络”访问，并确认没有开启会隔离局域网设备的访客 Wi-Fi。
5. 试玩结束后停止开发服务器，不要把这个低保真版本直接暴露到公网。

每台手机的进度与反馈只保存在该浏览器的 `localStorage`，不同手机不会自动汇总。

## 标准试玩流程

1. 让试玩者从首页开始，不先讲四维代码、权重或安全答案。
2. 记录开始时间；系统也会从点击“开始试玩”时自动计时。
3. 玩家完成前 11 题后进入平台更新终局，在 `/update` 完成第 12 题。
4. 玩家查看结果，再进入“填写试玩反馈”。
5. 反馈保存成功后，在同一页面导出 JSON 或 CSV。
6. 一名试玩者结束后，点击结果页“清除旧状态并重新测试”，再把设备交给下一名试玩者。

目标完成时间为 60—120 秒。不要提示玩家维度代码、分值或内部草案状态。

## 清除旧进度

首选方式：完成结果页后点击“清除旧状态并重新测试”。该操作会清除当前答案、计时和会话 ID，并立即创建新的匿名会话；已经本地保存的历史反馈记录会保留，方便批量导出。

如果玩家中途退出，需要彻底清空当前浏览器数据，可在浏览器站点设置中删除该站点的本地存储，或在开发者工具 Console 执行：

```js
localStorage.removeItem("aiti-playtest-v1")
location.assign("/")
```

彻底清空前应先导出已有反馈，否则本机尚未导出的反馈也会丢失。

## 导出反馈

反馈保存后会出现两个按钮：

- `playtest-results.json`：保留完整嵌套结构，适合归档和程序分析。
- `playtest-results.csv`：把 `answerTrace` 与 `feedback` 作为 JSON 字符串放入单元格，适合表格汇总。

每条记录只包含匿名会话 ID、时间戳、完成时长、答案轨迹、结果类型、哄感等级和反馈。不会写入姓名、手机号、微信身份、设备指纹、IP 或精确位置。自由评论只存在本地导出文件，不会自动上传。

## 汇总 5—15 份结果

1. 从每台试玩设备导出 JSON，按设备重命名文件，例如 `phone-a-playtest-results.json`，不要加入玩家姓名。
2. 将各文件中的记录数组合并，按 `anonymousSessionId` 去重；同一 ID 有多条反馈时保留时间戳最新的一条，并记录发生了重复提交。
3. 若使用 CSV，把各文件首行表头保留一次，再追加其他数据行。嵌套 JSON 字段不要手工拆逗号。
4. 计算描述性指标：完成时长中位数与范围、各题被选为最难/安全答案/不真实选项的次数、结果准确度与冒犯度的中位数、三项意愿的 yes 比例。
5. 逐条阅读自由评论，并按“题意不清、选项不对称、结果文案、流程问题、其他”归类。

样本只有 5—15 人时，任何百分比都只作内部产品信号。不要做显著性检验，不要外推总体，不要把类型比例或哄感等级比例解释为真实人群分布。

## 何时应修改题目

以下是本轮人工复核触发线，不是统计学阈值：

- 任一题被至少 3 人标为“一眼有安全答案”。
- 任一选项被至少 2 人标为“不像真人会选”。
- 任一题被至少三分之一试玩者选为最难，且评论集中在歧义或两项不可比。
- 多名玩家明确说舒服/可靠配对过于重复，或为了维持前后答案一致而修改选择。
- q10 或 q12 被多名玩家认为同时混入太多因素，印证其较高的哄感等级边际影响。
- 完成时长中位数超过 120 秒，或至少三分之一玩家无法在 120 秒内完成。
- 任一结果文案获得 4—5 分冒犯评分，且试玩者指出相同词句。
- 结果准确度中位数不高于 2，且评论显示定义难懂或结果与作答路径不符。
- 截图、了解研究或投票意愿中的任一项几乎无人选择 yes；先检查入口和文案，不把意愿解释为研究态度。

触发后应回到 `DRAFT_REQUIRES_HUMAN_REVIEW` 内容评审，逐项讨论并重新枚举、测试。不得仅根据少量试玩自动批准或自动改写题目。

## 调试页

开发环境可访问 `/debug`，用于查看答案轨迹、维度分、原始/归一化哄感分、平局信息和类型映射。不要向普通试玩者展示该页。生产构建会让 `/debug` 返回 404。
