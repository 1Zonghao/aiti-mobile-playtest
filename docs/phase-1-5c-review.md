# Phase 1.5C 交付复核

状态：低保真真人试玩版本完成。题目与哄感公式继续保持 `DRAFT_REQUIRES_HUMAN_REVIEW`，未自动批准。

## 实现范围

- 在原仓库增量初始化 Next.js 16 App Router、React 19、TypeScript strict、Tailwind CSS、Zustand、Vitest 与 Playwright。
- 保留并复用既有 `content/`、`src/scoring.ts`、Schema 和 Phase 1/1.5B 测试。
- 新增 `/`、`/test`、`/update`、`/result`、`/feedback` 与开发态 `/debug`。
- 前 11 题在 `/test` 一题一屏；原题库 q12 在 `/update` 作为第 12 题平台更新终局。没有添加额外评分题。
- 状态通过 Zustand 持久化到浏览器本地存储，刷新后从第一道未完成题恢复。
- 结果页显示类型、既定中文名称、定义、哄感等级、背离次数、致命台词、平台恐惧、毒舌总结、安全彩蛋和纯色几何占位角色。
- 反馈本地保存并可导出 `playtest-results.json` 与 `playtest-results.csv`。

## 内容与边界

- 16 种人格名称未修改。
- 普通页面不显示 `DRAFT_REQUIRES_HUMAN_REVIEW`、维度代码、分数、答案轨迹或平局信息。
- 结果页明确显示：“这不是人格诊断，也不预测真实心理依赖，只描述你在本次虚构互动中的选择路径。”
- `/debug` 仅在 `NODE_ENV=development` 时渲染；生产服务器黑盒访问返回 404，响应不含调试评分字段。
- 没有真实大模型调用、登录、账户、排行榜、社区、数据库或自动上传。
- 不采集姓名、手机号、微信身份、设备指纹、IP 或精确位置。

## 本地数据结构

每条导出记录包含：

- `anonymousSessionId`
- `timestamp`
- `completionTime`
- `answerTrace`
- `resultType`
- `temptationLevel`
- `feedback`

反馈中包含完成时长、类型、等级、背离次数、最难题、安全答案题、不真实选项题、准确度、冒犯度、截图/研究/投票意愿及可选评论。评论和其他记录只保存在当前浏览器，导出由用户主动触发。

## 可用性约束

- 页面容器以 480px 为上限，手机端优先。
- 全局正文基准 16px；主要按钮最小高度 48px，超过 44px 触控要求。
- 题目一次只显示一道，提供进度条与返回按钮。
- 选择后自动进入下一题；终局后自动评分。
- 重新测试会清空旧答案、计时和会话 ID，同时保留已保存反馈供汇总导出。
- 未实现正式插画或复杂动画；结果角色仅为内容配色驱动的几何占位。

## 已知限制与人工试玩重点

- 这是低保真版本，未测正式大会网络、二维码入口、分享卡生成或跨设备数据汇总。
- 反馈保存在各设备浏览器；清除站点数据前必须主动导出。
- 非首发人格的部分原始字段仍为 `MISSING`：结果标题未展示；缺少安全彩蛋时使用统一研究边界文案；缺少十六进制配色时使用中性几何占位色。没有把缺失内容伪造成已确认事实。
- q10/q12 对哄感等级变化频率较高，真人试玩应重点检查是否让玩家觉得混入过多判断因素。
- 5—15 人样本只用于发现题意、流程和文案问题，不能验证量表、估计人群分布或支持科学结论。
