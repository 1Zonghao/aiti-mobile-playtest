# Phase 2 设计系统

## 概念

视觉方向是“一本排版极其认真，但内容不太正经的 AI 关系人格档案”。界面使用奶白纸张、浅薄荷绿底色、墨绿规则线、低多边形角色、等宽档案标签与少量警示红。没有使用蓝紫 AI 渐变、玻璃拟态、霓虹、赛博朋克或企业仪表盘语言。

## 设计令牌

设计令牌统一定义于 `app/globals.css`：

| 用途 | 令牌 | 当前值 |
|---|---|---|
| 暖白背景 | `--paper` | `#f7f2e7` |
| 纸灰 | `--paper-deep` | `#e8e6dc` |
| 浅薄荷 | `--mint` | `#dceee1` |
| 深炭黑 | `--ink` | `#20231f` |
| 墨绿 | `--green` / `--rule` | `#173f35` / `#203b32` |
| 警示红 | `--warning` | `#d94b3d` |
| 信号橙 | `--signal` | `#e97a3c` |
| 记忆青 | `--memory` | `#39b8be` |

组件不手写主题颜色；角色专属配色来自 `content/result-types.json`。正式色值缺失的类型使用 V/F 两套稳定中性回退，不伪造确认配色。

## 排版与组件

- 系统中文字体栈，无远程阻塞字体。
- Impact/Arial Narrow 只用于巨型四字母代码和英文档案数字；正文保持系统中文字体。
- 主按钮最小高度 50px，选项最小高度 82px，移动端为 92px。
- 纸卡使用硬边框和实体错位阴影；无玻璃模糊堆叠。
- 交互状态同时使用文字、边框、阴影和颜色；不以颜色作为唯一信号。
- 所有可交互元素有可见 `focus-visible`。

## 动效与可访问性

选项选择后保留 300ms 反馈。平台更新使用卡片状态、记忆标签消散与轻微扫描错位，不使用强闪烁、音效、系统崩溃拟态或强制弹窗。`prefers-reduced-motion` 会把动画和延迟压缩到接近零，更新过程始终可跳过。

测试覆盖 360×640，页面结构支持 390×844、430×932 和 1366×768；全局禁止横向溢出，中文正文基准为 16px。
