# AITI Phase 1 交付复核

## 已完成

- 审计唯一 Markdown 内容源中的四维、16 型、8 个首发角色、配色、分享文案与视觉规范。
- 生成 7 个版本化 JSON，其中题目文件明确标记为草案。
- 建立严格 TypeScript 类型、Zod 运行时 Schema 和批量内容校验脚本。
- 实现纯函数四维评分、答案轨迹、稳定平局裁决与舒服—可靠背离计数。
- 实现与四字母类型独立的哄感等级算法草案。
- 建立 10 道题目草案，满足 3/2/3/2 维度覆盖、3 组配对和平台更新终局要求。
- 建立单元测试，覆盖 Markdown 16 型提取、唯一性、固定代码顺序、16 型可达性、确定性、平局、Schema、缺字段失败、免责声明、诊断措辞、等级边界和类型映射。

## 明确未完成

- 非首发 8 型缺少成品卡扩展文案和十六进制配色。
- 全部角色缺少实际图片路径和图片资产。
- 题目和哄感等级公式尚未人工验收或实证验证。
- 未开发页面、动画、分享图、数据库、登录、模型调用、数据采集或部署。

## 工程入口

- `npm run validate:content`：校验 7 个内容 JSON。
- `npm run lint`：静态代码检查。
- `npm run typecheck`：TypeScript strict 检查。
- `npm test`：Vitest 单元测试。

## Phase 1 停止条件

本阶段交付只包含内容结构化与评分引擎。后续进入页面开发前，应先由内容负责人验收题目草案、哄感公式、非首发缺失字段和视觉资产路径。
